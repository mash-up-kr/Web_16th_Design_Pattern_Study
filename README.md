# Design Pattern Study

웹 16기 디자인 패턴 스터디 레포지토리입니다.

## 인원

장정우, 윤신지, 이선민, 이창우, 김보경

## 범위

- https://patterns-dev-kr.github.io/design-patterns/introduction/
- **Design Patterns 파트만**

## 일정

- 매주 목요일 8~10시
- 디스코드 > 스터디

## 스터디 방식

1. 각자 자신의 **GitHub 아이디 브랜치**를 생성합니다.
2. 해당 브랜치에서 `1주차/이름/README.md` 형태로 폴더를 생성하여 정리한 내용을 올립니다.
3. `main` 브랜치로 **Pull Request**를 생성합니다. (PR 제목: `docs: [N주차] 이름 - 학습 주제`)
4. 스터디 완료 후 다른 사람의 PR에 코멘트를 작성합니다.
5. PR은 마지막에 확인(approve)한 사람이 머지(merge)를 진행해주세요.

## 스터디 규칙

- 같은 분량에 대해서 공부하고 [발표자](https://lazygyu.github.io/roulette/)를 정해서 발표
- 2시간을 넘어가지 않도록 한다
- 스터디 시작할 때 패턴별로 발표자 1명씩 정함
- 스터디 할 때 화면 캠 공유하기
- 한 패턴에 대한 발표를 끝내면 해당 패턴에 대한 논의 진행
  - `WHAT`, `HOW`, `WHY` 방식으로 질문을 준비
